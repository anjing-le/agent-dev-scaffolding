# Cursor 提示词范式目录

本目录包含了项目开发中常用的 Cursor 提示词模板，帮助快速生成符合项目规范的代码。

## 📁 目录结构

```
.cursor/prompts/
├── README.md                 # 本文档
├── api-module.md            # API 接口模块生成 Prompt
├── vue-list-page.md         # Vue 列表页面生成 Prompt
├── vue-business-component.md # Vue 业务组件生成 Prompt
└── vue-modal-form.md        # 新增/编辑弹窗生成 Prompt
```

## 🚀 使用方法

1. **选择合适的模板**：根据你的开发需求选择对应的 Prompt 文件
2. **复制模板内容**：将模板中的 Prompt 内容复制到 Cursor 聊天窗口
3. **填写具体信息**：根据提示填写具体的业务需求和参数
4. **生成代码**：让 Cursor 根据 Prompt 生成符合项目规范的代码

## 📋 模板列表

### 1. API 接口模块 (`api-module.md`)

- **用途**：生成完整的 API 接口服务类和类型定义
- **包含**：CRUD 接口、类型模型、JSDoc 注释
- **适用**：新建业务模块的 API 层开发

### 2. Vue 列表页面 (`vue-list-page.md`)

- **用途**：生成标准的数据列表页面
- **包含**：搜索表单、数据表格、操作按钮、分页功能
- **适用**：管理后台的列表页面开发

### 3. Vue 业务组件 (`vue-business-component.md`)

- **用途**：生成可复用的业务组件
- **包含**：Props/Emits 定义、逻辑封装、样式规范
- **适用**：公共组件或页面级组件开发

### 4. 新增/编辑弹窗 (`vue-modal-form.md`)

- **用途**：生成表单弹窗或抽屉组件
- **包含**：表单验证、数据绑定、提交逻辑
- **适用**：数据的新增和编辑功能

## 📝 自定义说明

### 占位符约定

- `[模块名称]`：业务模块的中文名称
- `[ModuleName]`：业务模块的 PascalCase 英文名称
- `[moduleName]`：业务模块的 camelCase 英文名称
- `[组件名称]`：组件的中文名称
- `[DataType]`：数据类型名称
